# 5. 数据结构设计

## 5.1 数据架构总览

### 5.1.1 三层数据模型

BidLens 的数据架构采用三层模型, 每一层服务于不同的处理阶段:

```
┌─────────────────────────────────────────────────────────────┐
│                    三层数据模型                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Layer 1: Raw Document (原始文档)                             │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ .docx (ZIP+XML) │ .pdf (二进制) │ .md (纯文本) │ ...   ││
│  │                  │               │               │       ││
│  │ 各格式私有的二进制/文本表示                               ││
│  └───────────────────────────┬─────────────────────────────┘│
│                              │ Parser Layer                  │
│                              ▼                               │
│  Layer 2: Document AST (文档抽象语法树)                       │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ 统一的 JSON 树形结构                                     ││
│  │ Section → Paragraph → TextRun                           ││
│  │         → Table → TableRow → TableCell                  ││
│  │         → List → ListItem                               ││
│  │         → Image, Header, Footer, Comment, ...           ││
│  │                                                         ││
│  │ 与源格式无关, 保留完整语义和样式信息                      ││
│  └───────────────────────────┬─────────────────────────────┘│
│                              │ Rust Engine                   │
│                              ▼                               │
│  Layer 3: Diff AST (比对结果抽象语法树)                       │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ Match 集合 (对齐后的节点配对)                            ││
│  │ Diff Detail 集合 (逐项差异描述)                          ││
│  │ Unmatched 集合 (未匹配节点)                              ││
│  │ Statistics (统计摘要)                                    ││
│  │ Metadata (元信息)                                        ││
│  └─────────────────────────────────────────────────────────┘│
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.1.2 数据流

```
.docx ──┐
.pdf  ──┼──► Parser ──► DocumentAST ──┐
.md   ──┘                             │
                                      ├──► Rust Engine ──► DiffAST ──► React UI
.docx ──┐                             │
.pdf  ──┼──► Parser ──► DocumentAST ──┘
.md   ──┘
```

### 5.2.15 节点属性继承与覆盖规则

```
┌─────────────────────────────────────────────────────┐
│              样式继承与覆盖规则                        │
├─────────────────────────────────────────────────────┤
│                                                      │
│  继承优先级 (从高到低):                               │
│                                                      │
│  1. 内联样式 (直接设置在节点上)                       │
│     例: TextRun.style.bold = true                   │
│     ↓ 覆盖                                           │
│  2. 段落样式 (Paragraph.style)                       │
│     例: Paragraph.style = "Heading1"                │
│     ↓ 继承自                                          │
│  3. 样式定义 (Styles.definitions)                    │
│     例: Heading1.basedOn = "Normal"                 │
│     ↓ 继承自                                          │
│  4. 默认样式 (内置默认值)                             │
│     例: fontFamily = "宋体", fontSize = 12           │
│                                                      │
│  合并策略:                                            │
│  ┌────────────────────────────────────────────────┐ │
│  │ 最终样式 = merge(                               │ │
│  │   defaults,          // 默认值                  │ │
│  │   styleChain...,     // 样式继承链 (从远到近)   │ │
│  │   paragraph.style,   // 段落直接样式            │ │
│  │   inline.style       // 行内直接样式            │ │
│  │ )                                               │ │
│  └────────────────────────────────────────────────┘ │
│                                                      │
│  特殊规则:                                            │
│  - 字体: 直接覆盖, 不继承 (西文字体/中文字体分开)     │
│  - 颜色: 支持 "auto" 值, 表示使用父级颜色            │
│  - 行距: 支持 "exact"(固定值) 和 "atLeast"(最小值)  │
│  - 边框: 不继承, 每个节点独立定义                     │
│                                                      │
└─────────────────────────────────────────────────────┘
```

### 5.2.16 样式合并算法

```typescript
// ast/style-merge.ts
interface StyleChain {
  defaults: BaseStyle;
  definitions: Map<string, StyleDefinition>;
  paragraphStyleName?: string;
  paragraphOverrides: Partial<BaseStyle>;
  inlineOverrides: Partial<BaseStyle>;
}

function resolveStyle(chain: StyleChain): ResolvedStyle {
  // 1. 从默认值开始
  let resolved: Record<string, any> = { ...chain.defaults };

  // 2. 沿继承链向上合并
  let currentStyleName = chain.paragraphStyleName;
  const visited = new Set<string>();

  while (currentStyleName && !visited.has(currentStyleName)) {
    visited.add(currentStyleName);
    const def = chain.definitions.get(currentStyleName);
    if (def) {
      resolved = { ...resolved, ...def.properties };
      currentStyleName = def.basedOn;
    } else {
      break;
    }
  }

  // 3. 合并段落直接样式
  resolved = { ...resolved, ...chain.paragraphOverrides };

  // 4. 合并行内直接样式
  resolved = { ...resolved, ...chain.inlineOverrides };

  return resolved as ResolvedStyle;
}
```

### 5.3.1 路径式 ID 格式规范

节点 ID 采用路径式命名, 体现节点在文档树中的位置关系:

```
格式: doc::{type}{index}[::{type}{index}]...

类型缩写:
  s    = section
  p    = paragraph
  r    = text_run
  t    = table
  tr   = table_row
  tc   = table_cell
  img  = image
  list = list
  li   = list_item
  hl   = hyperlink
  fc   = field_code
  eq   = equation
  bm   = bookmark
  pb   = page_break
  hdr  = header
  ftr  = footer
  fn   = footnote
  en   = endnote
  cmt  = comment
  rev  = revision

示例:
  doc                           → 文档根节点
  doc::s1                       → 第1个 Section
  doc::s1::p3                   → 第1个 Section 的第3个 Paragraph
  doc::s1::p3::r2               → 第3个 Paragraph 的第2个 TextRun
  doc::s1::t1::tr2::tc3         → 第1个 Table 的第2行第3列
  doc::s1::p5::img1             → 第5个 Paragraph 中的第1个 Image
  doc::s1::list1::li2::p1::r1   → 第1个 List 的第2个 ListItem 的第1个 Paragraph 的第1个 TextRun
```

### 5.3.2 ID 分配算法

```typescript
// ast/id-allocator.ts
class NodeIdAllocator {
  private counters = new Map<string, number>();

  allocate(parentPath: string, type: string): string {
    const key = `${parentPath}::${type}`;
    const count = (this.counters.get(key) ?? 0) + 1;
    this.counters.set(key, count);
    return `${key}${count}`;
  }

  /**
   * 批量分配 (用于表格等规则结构)
   * 例如: allocateGrid("doc::s1::t1", "tr", 5, "tc", 4)
   * 返回二维数组: [[tr1::tc1, tr1::tc2, ...], [tr2::tc1, ...], ...]
   */
  allocateGrid(
    parentPath: string,
    rowType: string, rowCount: number,
    colType: string, colCount: number
  ): string[][] {
    const grid: string[][] = [];
    for (let r = 0; r < rowCount; r++) {
      const rowPath = this.allocate(parentPath, rowType);
      const row: string[] = [];
      for (let c = 0; c < colCount; c++) {
        row.push(this.allocate(rowPath, colType));
      }
      grid.push(row);
    }
    return grid;
  }

  reset(): void {
    this.counters.clear();
  }
}
```

### 5.3.3 ID 解析与遍历

```typescript
// ast/id-utils.ts
interface ParsedId {
  segments: Array<{ type: string; index: number }>;
  depth: number;
  parentPath: string | null;
  nodeType: string;
  nodeIndex: number;
}

function parseNodeId(id: string): ParsedId {
  const segments = id.split('::').map(seg => {
    const match = seg.match(/^([a-z]+)(\d+)$/);
    if (!match) throw new Error(`Invalid ID segment: ${seg}`);
    return { type: match[1], index: parseInt(match[2], 10) };
  });

  return {
    segments,
    depth: segments.length - 1,
    parentPath: segments.length > 1
      ? segments.slice(0, -1).map(s => `${s.type}${s.index}`).join('::')
      : null,
    nodeType: segments[segments.length - 1].type,
    nodeIndex: segments[segments.length - 1].index,
  };
}

// 遍历 AST 时, 利用 ID 前缀快速判断祖先关系
function isAncestor(ancestorId: string, descendantId: string): boolean {
  return descendantId.startsWith(ancestorId + '::');
}
```

## 5.4 Diff AST 完整 Schema

### 5.4.1 顶层结构

```typescript
interface DiffAST {
  /** 差异结果 ID */
  diffId: string;

  /** 文档 A (基准) 的 SHA256 哈希 — 通过此 ID 从 document_cache 获取 DocumentAST */
  docAHash: string;

  /** 文档 B (比较) 的 SHA256 哈希 — 通过此 ID 从 document_cache 获取 DocumentAST */
  docBHash: string;

  /** 匹配集合 (对齐后的节点配对) */
  matches: Match[];

  /** 未匹配集合 (仅存在于 A 或 B 的节点) */
  unmatched: Unmatched[];

  /** 统计信息 */
  statistics: DiffStatistics;

  /** 元信息 */
  metadata: DiffMetadata;
}

// 注意：DiffAST 不嵌入完整的 DocumentAST。前端需要原始文档内容时，
// 通过 docAHash / docBHash 从 Electron 主进程的 document_cache 中获取。
// 这避免了 DiffAST BLOB 的体积膨胀（DocumentAST 可达 50-100MB）。

interface DiffMetadata {
  /** 比对开始时间 */
  startedAt: string;

  /** 比对结束时间 */
  completedAt: string;

  /** 比对耗时 (ms) */
  duration: number;

  /** 使用的比对引擎版本 */
  engineVersion: string;

  /** 比对配置 */
  config: CompareConfig;

  /** 节点匹配算法 */
  matchAlgorithm: string;

  /** 文本差异算法 */
  diffAlgorithm: string;
}
```

### 5.4.2 Match 结构

> **Rust 类型映射**：前端 `Match` 对应 Rust 的 `NodeDiff::Identical` / `NodeDiff::Modified` / `NodeDiff::Moved`。
> 前端 `Unmatched` 对应 Rust 的 `NodeDiff::Added` / `NodeDiff::Removed`。
> 转换逻辑在 `DiffAstBuilder` 中实现（参见文档02第2.3.8节）。

```typescript
interface Match {
  /** 匹配 ID */
  matchId: string;

  /**
   * 匹配类型（差异结果维度）
   * - 'identical': 完全相同（Rust: NodeDiff::Identical）
   * - 'modified': 内容/格式有差异（Rust: NodeDiff::Modified）
   * - 'style_only': 仅样式差异（Rust: NodeDiff::Modified + Change::StyleChanged）
   * - 'reordered': 位置调整（Rust: NodeDiff::Moved）
   */
  matchType: 'identical' | 'modified' | 'style_only' | 'reordered';

  /** 匹配置信度 [0, 1] */
  confidence: number;

  /** 语义相似度 [0, 1] */
  similarity: number;

  /** 匹配分类 (用于 UI 分组) */
  matchCategory: 'text' | 'style' | 'table' | 'comment' | 'revision' | 'image' | 'format';

  /** 文档 A 中的节点 ID */
  sourceA: string;

  /** 文档 B 中的节点 ID */
  sourceB: string;

  /** 差异详情 (identical 类型时为空) */
  diff?: DiffDetail;

  /** 匹配算法提供的额外信息 */
  matchInfo?: {
    /**
     * 使用的匹配策略（算法来源维度，区别于 matchType 的结果维度）
     * - 'exact': 完全匹配（Rust: MatchType::Exact）
     * - 'semantic': 语义匹配（Rust: MatchType::Semantic）
     * - 'fuzzy': 文本模糊匹配（Rust: MatchType::Textual）
     * - 'structural': 结构匹配（Rust: MatchType::Structural）
     */
    strategy: 'exact' | 'fuzzy' | 'semantic' | 'structural';

    /** 匹配的特征向量 */
    features?: string[];
  };
}
```

## 5.7 数据校验

### 5.7.1 AST 完整性校验

```typescript
// ast/validator.ts
class ASTValidator {
  validate(ast: unknown): ValidationResult {
    const errors: ValidationError[] = [];
    const warnings: ValidationWarning[] = [];

    // 1. 基础结构检查
    if (!ast || typeof ast !== 'object') {
      errors.push({ code: 'E001', message: 'AST 根节点不是对象' });
      return { valid: false, errors, warnings };
    }

    const doc = ast as DocumentAST;

    // 2. 必填字段检查
    this.checkRequired(doc, ['id', 'type', 'version', 'sections'], errors);

    // 3. 类型检查
    if (doc.type !== 'document') {
      errors.push({ code: 'E002', message: `根节点类型应为 "document", 实际为 "${doc.type}"` });
    }

    // 4. 递归校验所有子节点
    for (const section of doc.sections ?? []) {
      this.validateNode(section, 'doc', errors, warnings);
    }

    // 5. ID 唯一性检查
    this.checkIdUniqueness(doc, errors);

    // 6. 异常值检测
    this.checkAnomalies(doc, warnings);

    return {
      valid: errors.length === 0,
      errors,
      warnings,
    };
  }

  private validateNode(
    node: any,
    parentPath: string,
    errors: ValidationError[],
    warnings: ValidationWarning[]
  ): void {
    // 必填字段
    if (!node.id) errors.push({ code: 'E003', message: `节点缺少 id 字段`, location: parentPath });
    if (!node.type) errors.push({ code: 'E004', message: `节点 ${node.id} 缺少 type 字段` });

    // ID 格式检查
    if (node.id && !node.id.startsWith(parentPath + '::') && parentPath !== 'doc') {
      errors.push({ code: 'E005', message: `节点 ID "${node.id}" 与父节点路径不匹配` });
    }

    // TextRun 文本检查
    if (node.type === 'text_run') {
      if (typeof node.text !== 'string') {
        errors.push({ code: 'E006', message: `TextRun ${node.id} 的 text 字段不是字符串` });
      }
    }

    // 递归检查子节点
    if (node.children && Array.isArray(node.children)) {
      for (const child of node.children) {
        this.validateNode(child, node.id, errors, warnings);
      }
    }
  }

  private checkAnomalies(doc: DocumentAST, warnings: ValidationWarning[]): void {
    this.walkTree(doc, (node) => {
      // 空文本警告
      if (node.type === 'text_run' && node.text === '') {
        warnings.push({ code: 'W001', message: `TextRun ${node.id} 文本为空`, nodeId: node.id });
      }

      // 异常字号
      if (node.style?.fontSize && (node.style.fontSize < 1 || node.style.fontSize > 200)) {
        warnings.push({ code: 'W002', message: `节点 ${node.id} 字号异常: ${node.style.fontSize}pt`, nodeId: node.id });
      }

      // 嵌套层级过深
      const depth = node.id.split('::').length;
      if (depth > 10) {
        warnings.push({ code: 'W003', message: `节点 ${node.id} 嵌套层级过深 (${depth})`, nodeId: node.id });
      }

      // 图片尺寸异常
      if (node.type === 'image') {
        if (node.width > 10000 || node.height > 10000) {
          warnings.push({ code: 'W004', message: `图片 ${node.id} 尺寸异常: ${node.width}×${node.height}`, nodeId: node.id });
        }
      }
    });
  }

  private checkIdUniqueness(doc: DocumentAST, errors: ValidationError[]): void {
    const seen = new Set<string>();
    this.walkTree(doc, (node) => {
      if (seen.has(node.id)) {
        errors.push({ code: 'E007', message: `节点 ID "${node.id}" 重复` });
      }
      seen.add(node.id);
    });
  }
}
```

### 5.7.2 Diff AST 校验

```typescript
class DiffASTValidator {
  validate(diff: DiffAST, docA: DocumentAST, docB: DocumentAST): ValidationResult {
    const errors: ValidationError[] = [];
    const warnings: ValidationWarning[] = [];

    // 1. 引用完整性: 所有 sourceA/sourceB 必须指向存在的节点
    for (const match of diff.matches) {
      if (!this.nodeExists(docA, match.sourceA)) {
        errors.push({ code: 'D001', message: `Match ${match.matchId} 引用了不存在的 docA 节点: ${match.sourceA}` });
      }
      if (!this.nodeExists(docB, match.sourceB)) {
        errors.push({ code: 'D002', message: `Match ${match.matchId} 引用了不存在的 docB 节点: ${match.sourceB}` });
      }
    }

    // 2. 无重复引用: 每个节点最多被匹配一次
    const matchedA = new Set<string>();
    const matchedB = new Set<string>();
    for (const match of diff.matches) {
      if (matchedA.has(match.sourceA)) {
        errors.push({ code: 'D003', message: `docA 节点 ${match.sourceA} 被多次匹配` });
      }
      if (matchedB.has(match.sourceB)) {
        errors.push({ code: 'D004', message: `docB 节点 ${match.sourceB} 被多次匹配` });
      }
      matchedA.add(match.sourceA);
      matchedB.add(match.sourceB);
    }

    // 3. 统计一致性
    const expectedMatches = diff.statistics.totalMatches;
    const actualMatches = diff.matches.length;
    if (expectedMatches !== actualMatches) {
      errors.push({ code: 'D005', message: `统计 totalMatches (${expectedMatches}) 与实际 matches 数量 (${actualMatches}) 不一致` });
    }

    // 4. 相似度范围检查
    for (const match of diff.matches) {
      if (match.confidence < 0 || match.confidence > 1) {
        warnings.push({ code: 'DW001', message: `Match ${match.matchId} 置信度超出范围: ${match.confidence}` });
      }
      if (match.similarity < 0 || match.similarity > 1) {
        warnings.push({ code: 'DW002', message: `Match ${match.matchId} 相似度超出范围: ${match.similarity}` });
      }
    }

    return { valid: errors.length === 0, errors, warnings };
  }
}
```

> **本章小结**: BidLens 的数据架构采用三层模型 (Raw Document → DocumentAST → DiffAST), 通过 20+ 种节点类型覆盖了 Word 文档的完整语义。路径式节点 ID 系统 (`doc::s1::p3::r2`) 既体现了文档层级结构, 又支持高效的祖先关系判断。DiffAST 通过 Match/Unmatched/Statistics 三部分完整描述了两份文档的差异。Embedding 向量使用 Float32 BLOB 存储, 相似度矩阵支持大文档的分块计算。JSON 序列化配合 gzip 压缩实现了 83-86% 的压缩率。完善的数据校验体系从解析、缓存、比对到渲染四个环节保障了数据完整性。
