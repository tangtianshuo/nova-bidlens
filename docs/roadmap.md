# 版本路线图

> 版本：v2.0 | 最后更新：2026-07-21

## 一、版本列表

| 版本 | 主题 | 状态 | 关键特性 |
|------|------|------|----------|
| **V0.1** | 语义比对原型 | ✅ 完成 | 段落级比对、三栏工作台（功能原型） |
| **V0.2** | 全保真原型 | ✅ 完成 | 表格、格式、批注、PDF（功能原型） |
| **V0.2.1** | 原型稳定性 | ✅ 完成 | 文档解析增强、Rust引擎优化（功能原型） |
| **V0.2.2** | 原型生产验证 | ✅ 完成 | 真实流水线、持久化、加密、设计系统（功能原型验证） |

> 这些能力作为 V0.3 雷同性风险审查的基础设施继续复用。双文档比对不再作为独立产品入口，底层 Diff 能力保留为 RiskFinding 和文件对的深入证据工具。

## 六、V0.3 — 雷同性风险审查 🔄

**权威产品依据：** `docs/product/PRD-v0.3-similarity-risk-review.md`

**目标：** 交付本地化、可解释的投标文件雷同性风险审查工作台。

**产品定位：** 以招标项目为单位，导入 2-8 份投标文件和可选招标基线，执行跨文件语义、表格和实体检测，输出项目风险、文件关系矩阵和可追溯证据。

### 6.1 V0.3.0：基础雷同性审查闭环

**产品和 UI 基础：**

- [x] 产品主干和权威 PRD。
- [x] 风险项目 Shared 类型和 `risk:*` IPC 初版。
- [x] 风险设计令牌、项目页面、关系矩阵、发现项和证据复核 UI。
- [x] 响应式、无障碍和 1000+ Finding 前端性能验证。
- [ ] 按 PRD 拆分 ProjectStatus、AnalysisPhase 和 SubmissionProcessingState。
- [ ] 移除独立双文档产品入口，将 Diff 接入 RiskFinding 证据链。

**真实项目主链：**

- [ ] 2-8 份文件选择、校验、DOCX/PDF 解析和初步 `risk:*` IPC 已部分接线，尚未完成端到端验证。
- [ ] 通用 ReviewNode、业务标签、实体和关键事实提取。
- [ ] 招标公共内容过滤。
- [ ] 内容 Hash、字符 n-gram、实体/事实、表格签名四路稀疏召回。
- [ ] 文本、表格、实体和关键事实四类检测器。
- [ ] 跨检测器聚合去重、文件对风险和项目风险。
- [ ] 项目、AST、ReviewNode、Evidence、复核和检查点加密持久化。
- [ ] 取消、中断、Partial 和兼容检查点恢复。
- [ ] 人工复核真实保存和 PDF/HTML/Markdown 项目报告。
- [ ] 真实文件 Electron E2E 和正式构建 Fixture 隔离验证。

### 6.2 V0.3.1：本地语义增强

- [ ] BGE-M3 模型许可和运行证据门槛通过。
- [ ] 本地 ONNX Provider 和模型包生命周期。
- [ ] 稳定 Chunk、加密向量缓存和缓存失效规则。
- [ ] 项目内语义 Top-K 召回。
- [ ] 语义、词法、结构、实体和事实混合精排。
- [ ] 模型不可用、损坏、OOM 和 Provider 崩溃降级路径。

### 6.3 V0.3.2：质量与发布加固

- [ ] 授权或脱敏金标数据集和盲测。
- [ ] 严格、标准、宽松预设校准。
- [ ] 混合语义匹配 F1 相对基线提升至少 15 个百分点。
- [ ] 向量候选 Recall@TopK 不低于 95%。
- [ ] 明显错误匹配率不高于 5%。
- [ ] 4000 页项目资源、安全和恢复测试。
- [ ] 断网安装、模型升级回滚、应用签名和安装验证。

### 6.4 能力迁移原则

- 双文档版本比对不再是独立产品入口。
- `DiffAst`、表格 Diff、格式、批注和修订能力保留为证据工具。
- `compare:*` 在 RiskFinding 证据链完成迁移前保留兼容性，不得先行删除。
- 企业知识库和 AI 编标只保留架构兼容性，不进入 V0.3 验收。
