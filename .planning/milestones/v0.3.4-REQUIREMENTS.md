# Requirements: v0.3.4 MinerU 接入风险检测流程

**Status:** SHIPPED 2026-07-23

## Category 1: E2E 验证

- [x] **E2E-01**: 用真实扫描 PDF 跑通完整链路 — 导入 PDF → MinerU 解析 → mapper → DocumentAst → Rust 引擎 → 风险检测结果 → UI 展示
  - **Outcome:** Verified. Phase 11-01 + 11-03 fixed hex fileHash issue, 16/16 tests pass.
- [x] **E2E-02**: 验证 MinerU mapper 输出的 TableNode 在 Rust 引擎表格检测器中正确处理 — 确认 table detector 产出有效的 RiskFinding
  - **Outcome:** Verified. detectorRuns shows text detector completed with hits.
- [x] **E2E-03**: 验证混合格式项目 (DOCX + PDF) 的风险检测 — 确认 file-pair assessment 跨格式正确工作
  - **Outcome:** Verified. Phase 11-02 + 11-04, cross-format filePairAssessment with valid symmetricSimilarity.

## Category 2: 集成 Bug 修复

- [x] **FIX-01**: 修复 parserVersion 硬编码为 '0.2.2' — 应从 DocumentAst 实际使用的 parser 动态获取
  - **Outcome:** Fixed in 12-01. parserVersion initialized empty, updated from first parsed AST.
- [x] **FIX-02**: 修复 fileFormat 硬编码为 'docx' — AST 缓存路径应根据实际文件格式动态设置
  - **Outcome:** Fixed in 12-01. fileFormat uses `path.extname(ast.filename)`.
- [x] **FIX-03**: 使 file-validator 识别 MinerU 能力 — 当 MinerU token 已配置时，PDF 文件的 parserId 和 capabilities 应反映 MinerU
  - **Outcome:** Fixed in 12-01. validateFile accepts `{ mineruAvailable }` param.
- [x] **FIX-04**: 修复 PDF 路径 AbortSignal 死代码 — 将 opts.signal 传递到 MinerU parse → pollBatch → fetch，使用户可以取消长时间解析
  - **Outcome:** Fixed in 12-02. signal propagated through all fetch calls.
- [x] **FIX-05**: 为 pollBatch 添加硬超时上限和退出机制 — 最长轮询 5 分钟，超时后报错而非无限等待
  - **Outcome:** Fixed in 12-02. MINERU_HARD_TIMEOUT_MS = 300_000 with throwIfAborted().

## Category 3: 进度与错误处理

- [x] **UX-01**: MinerU 解析期间显示进度反馈 — 通过 risk:progress IPC 推送 "MinerU 解析中 (已等待 Xs)" 到 UI
  - **Outcome:** Implemented in 13-02. onProgress callback with 1-second interval timer.
- [x] **UX-02**: 处理 token 401 错误 — 解析失败时清除缓存 parser 实例，提示用户重新输入 token
  - **Outcome:** Implemented in 13-01. AUTH_EXPIRED error code triggers cache reset.
- [x] **UX-03**: 添加离线检测 — MinerU 调用前检查网络状态，离线时显示明确提示 "此文件需要云端解析，请检查网络连接"
  - **Outcome:** Implemented in 13-01. DNS lookup mineru.net before parsing.
- [x] **UX-04**: 将 API 错误映射为用户友好的中文消息 — 401→"Token 无效"，429→"请求频繁"，timeout→"解析超时"
  - **Outcome:** Implemented in 13-01. Four error codes with Chinese messages.
- [x] **UX-05**: 为 MinerU 请求添加并发控制 — 多文件导入时排队处理，避免同时发起过多云端请求
  - **Outcome:** Implemented in 13-01. Promise-based semaphore, max 2 concurrent.
