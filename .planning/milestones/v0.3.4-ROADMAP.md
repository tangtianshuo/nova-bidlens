**Depends on**: Phase 10 (v0.3.3 调研完成)
**Plans**: 4 plans (2 initial + 2 gap closure)

Plans:

- [x] 11-01: PDF 全链路 E2E 验证（MinerU → mapper → AST → Rust 引擎 → RiskFinding）
- [x] 11-02: 混合格式项目（DOCX + PDF）跨格式风险检测验证
- [x] 11-03: [Gap] 修复 fileHash 为合法 hex 字符串，使引擎产出 findings
- [x] 11-04: [Gap] 修复 mixed-format fileHash + 增强 detectorRuns 验证

**Details:**
- 验证 12/12 truths (初始 8/12，gap closure 修复剩余 4 个)
- 发现根因：测试 fileHash 使用非法 hex 字符串 → hex::decode 返回空 bytes → 相同 node ID → 0 findings
- 引擎代码无 bug，问题完全在测试数据
- 137/137 集成测试通过

### Phase 12: 集成 Bug 修复

**Goal**: 修复管道中的硬编码值、死代码和缺失能力声明，使 MinerU 路径行为正确
**Depends on**: Phase 11
**Plans**: 2 plans

Plans:

- [x] 12-01: 修复 parserVersion/fileFormat 硬编码 + file-validator MinerU 能力识别 (FIX-01, FIX-02, FIX-03)
- [x] 12-02: MinerU AbortSignal 传播 + pollBatch 硬超时 (FIX-04, FIX-05)

**Details:**
- parserVersion 从解析后的 AST 动态获取，不再硬编码 '0.2.2'
- fileFormat 从 `path.extname(filename)` 动态获取，不再硬编码 'docx'
- file-validator 识别 MinerU 能力：token 已配置时 PDF 返回 'mineru-parser'
- AbortSignal 从 parser-service → MinerU parse → uploadFile → pollBatch → downloadResult 全链路传播
- pollBatch 硬超时 300s（5 分钟），超时后报错而非无限等待

### Phase 13: UX 打磨

**Goal**: MinerU 解析体验达到生产质量——有进度反馈、友好错误提示、离线检测和并发控制
**Depends on**: Phase 12
**Plans**: 2 plans

Plans:

- [x] 13-01: 错误处理、离线检测、Token 清除、并发控制 (UX-02, UX-03, UX-04, UX-05)
- [x] 13-02: MinerU 进度反馈链路 (UX-01)

**Details:**
- MinerU 解析期间 UI 显示 "MinerU 解析中 (已等待 Xs)"，1 秒更新
- 401 错误自动清除缓存 parser 实例，返回 AUTH_EXPIRED 错误码
- 离线检测：DNS lookup mineru.net，失败时提示 "此文件需要云端解析，请检查网络连接"
- 错误码映射：AUTH_EXPIRED→"Token 无效"，RATE_LIMITED→"请求频繁"，MINERU_TIMEOUT→"解析超时"，MINERU_OFFLINE→"网络不可用"
- 并发控制：promise-based semaphore，最多 2 个并发 MinerU 请求

## Milestone Summary

**Key Decisions:**
- fileHash 必须使用合法 64 字符 hex 字符串（SHA256 格式），否则引擎 hex::decode 返回空 bytes
- AbortSignal 通过 ParseOptions.signal 传递，不新增 IPC 通道
- onProgress 回调通过 ParseOptions 传递，MinerU parser 内部启动定时器
- 并发控制用简单 counter + queue，不用外部库
- 离线检测用 node:dns/promises lookup，轻量无依赖

**Issues Resolved:**
- Engine returns 0 findings for identical submissions (root cause: invalid hex fileHash in tests)
- parserVersion hardcoded to '0.2.2' regardless of actual parser
- fileFormat hardcoded to 'docx' for all files
- AbortSignal not propagated to MinerU API calls
- pollBatch has no timeout ceiling
- No progress feedback during 1-3 minute MinerU cloud parsing
- 401 errors leave stale parser cache
- No offline detection for cloud-dependent parsing
- No concurrency control for multiple MinerU requests

**Technical Debt:**
- parser-service.test.ts fails in test env (Electron ipcMain not mocked) — pre-existing, unrelated
- Evidence validation in mixed-format test is conditional (findings.length === 0 is normal for different content)
