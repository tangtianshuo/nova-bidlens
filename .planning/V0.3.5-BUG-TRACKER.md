# v0.3.5 Bug Tracker

**用途:** 人机协同测试期间登记发现的 bug 和改进项。Claude 读取此文件进行 v0.3.5 里程碑规划。

**使用方式:**
1. 按下方测试清单逐项验证
2. 发现问题时在对应分类下添加条目
3. 格式: `- [ ] BUG-XX: 问题描述 | 严重程度(critical/major/minor) | 复现步骤`

---

## 功能测试清单

### 1. 项目创建与文件导入

- [ ] 创建新项目，设置项目名称
- [ ] 导入 1 个 DOCX 文件，确认解析成功
- [ ] 导入 1 个数字版 PDF，确认解析成功
- [ ] 导入 1 个扫描版 PDF，确认 MinerU 解析成功（需网络 + token）
- [ ] 导入 nZBTF 文件（ZIP/XML），确认解析成功
- [ ] 导入不支持的文件格式，确认有友好提示
- [ ] 重复导入同一文件，确认行为正确
- [ ] 同时导入 2+ 文件，确认并发控制生效

### 2. 风险检测流程

- [ ] 2 个 DOCX 文件提交分析，确认 findings 产出
- [ ] 2 个 PDF 文件提交分析，确认 findings 产出
- [ ] 混合格式（DOCX + PDF）提交分析，确认跨格式检测
- [ ] 相同内容的文件，确认高相似度 findings
- [ ] 完全不同的文件，确认低相似度或无 findings
- [ ] 分析进度显示正常（已等待秒数）
- [ ] 分析过程中可以取消

### 3. 结果展示

- [ ] 风险矩阵正确显示 file-pair assessments
- [ ] Findings 列表显示检测类型、风险等级
- [ ] 点击 finding 可查看详情
- [ ] Evidence 显示原始文本引用
- [ ] Evidence 链接到正确的 submission
- [ ] 表格检测结果正确显示

### 4. 错误处理

- [ ] MinerU token 无效时，提示 "Token 无效"
- [ ] 网络离线时，提示 "此文件需要云端解析，请检查网络连接"
- [ ] API 请求频繁时，提示 "请求频繁"
- [ ] 解析超时时，提示 "解析超时"
- [ ] Token 过期（401）时自动清除缓存并提示重新输入
- [ ] 引擎崩溃时有错误提示而非静默失败

### 5. UI/UX

- [ ] 中文界面显示正确，无乱码
- [ ] 加载状态有明确视觉反馈
- [ ] 错误消息用户友好且为中文
- [ ] 页面布局在不同窗口大小下正常
- [ ] 所有按钮可点击且有响应
- [ ] 导航流程清晰

### 6. 设置与配置

- [ ] MinerU token 设置界面可用
- [ ] Token 保存后可正常使用
- [ ] Token 修改后生效
- [ ] 设置页面其他选项可用
