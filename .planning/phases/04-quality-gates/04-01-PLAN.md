Create `tests/security/security.test.ts` with the following test groups:

**1. Offline operation (no network imports)**
- Read `apps/desktop/src/main/index.ts` and all files in `apps/desktop/src/main/` recursively
- Parse imports and verify no `fetch`, `http`, `https`, `net`, `node:http`, `node:https`, `axios`, `got`, `node-fetch` imports exist
- Use a simple AST approach: read file contents, regex match import statements
- Test: `it('main process has no network imports')` — glob all .ts files in src/main/, grep for network-related imports, expect none

**2. Encrypted DB/WAL verification**
- Import `encrypt` and `decrypt` from `apps/desktop/src/main/db/crypto.ts`
- Test roundtrip: encrypt a string with a random 32-byte key, decrypt it, verify equality
- Test: encrypt produces different ciphertext for same plaintext (random IV)
- Test: decrypt with wrong key throws error
- Test: decrypt with truncated payload throws error (payload < IV_LENGTH + TAG_LENGTH = 28 bytes)
- Import schema SQL from `apps/desktop/src/main/db/schema.ts`
- Test: `CREATE_TABLES_SQL` contains `payload_encrypted BLOB` columns (for document_snapshots, diff_snapshots)
- Test: `CREATE_TABLES_SQL` contains `note_encrypted BLOB` (for review_annotations)
- Test: schema sets `journal_mode = WAL` pragma (check in database.ts — the `pragma('journal_mode = WAL')` call exists)
- Test: schema sets `ENABLE_FOREIGN_KEYS_SQL` is imported and used

**3. Deletion closure**
- Create a test that verifies the delete SQL cascade works
- Since we can't easily spin up better-sqlite3 in vitest/jsdom, test at the repository level:
  - Mock or use an in-memory SQLite approach
  - OR: verify the SQL statements reference `ON DELETE CASCADE` in schema.ts
  - Pragmatic approach: parse CREATE_TABLES_SQL and verify foreign keys with ON DELETE CASCADE exist for document_snapshots, diff_snapshots, review_annotations

**4. Console.log safety (no key logging)**
- Read all .ts files in `apps/desktop/src/main/`
- Grep for console.log/console.error/console.warn calls
- Verify none of them log variables named `key`, `masterKey`, `encryptionKey`, `plaintext`, `password`, `secret`
- Test: `it('main process does not log encryption keys')` — static analysis of console.log calls
