**测试组：混合格式风险分析**

1. `it('DOCX and PDF produce valid DocumentAst independently')`
   - 验证 docxAst.blocks 非空
   - 验证 pdfAst.blocks 非空
   - 验证两者的 parserVersion 不同（'docx4js' vs 'mineru-api-v4'）

2. `it('toEngineDocumentAst converts both formats correctly')`
   - 调用 `toEngineDocumentAst(docxAst)` 和 `toEngineDocumentAst(pdfAst)`
   - 验证两者都有 blocks 数组
   - 验证两者的 parser_version 字段保留了原始值

3. `it('Rust engine processes mixed-format project and produces cross-format finding')`
   - 启动 Rust 引擎进程
   - 发送 risk.analyzeWithAst 请求，submissions 包含两个不同格式的 AST：
     ```
     submissions: [
       { submissionId: 'sub-docx', fileHash: docxAst.sha256, ast: toEngineDocumentAst(docxAst) },
       { submissionId: 'sub-pdf', fileHash: pdfAst.sha256, ast: toEngineDocumentAst(pdfAst) }
     ]
     ```
   - 验证响应：
     - `findings` 数组存在（可能为空——不同文档内容不同，这是正常的）
     - `filePairAssessments` 数组存在且长度 >= 1
     - filePairAssessment 中 `submissionAId` 和 `submissionBId` 分别对应 'sub-docx' 和 'sub-pdf'
     - `symmetricSimilarity` 在 [0, 1] 范围内
     - `riskLevel` 是 'low'、'medium' 或 'high' 之一
   - 如果 findings 非空：
     - 验证 finding 的 `involvedSubmissionIds` 包含 'sub-docx' 和/或 'sub-pdf'
     - 验证 evidence 的 `sourceSubmissionId` 和 `targetSubmissionId` 引用正确的 submission

4. `it('cross-format project risk assessment is computed')`
   - 从同一引擎响应中验证 `projectRisk` 存在
   - `projectRisk.level` 是 'low'、'medium' 或 'high'
   - `projectRisk.rawRuleScore` 是数字

**引擎通信模式：**

- 引擎二进制路径同 11-01
