<!-- Root cause analysis -->
<root_cause>
测试中 `fileHash` 使用了 `'sha256-scanned-test'` 和 `'sha256-scanned-test-2'` 这类非 hex 字符串。
引擎 Rust 端 `hex::decode(&sub.file_hash).unwrap_or_default()` 对非 hex 字符串返回空 `Vec<u8>`。
`generate_node_id(file_hash, &[node_index])` 对两个 submission 产生相同 node ID（因为 file_hash 都是空的）。
TextDetector 的 node_map 只保留每个 node ID 的最后一个节点，导致比较的是同一 submission 内的节点。
aggregation 跳过 source_submission_id == target_submission_id 的 evidence → findings = 0。

修复方案：使用合法的 64 字符 hex 字符串作为 fileHash。
</root_cause>
