Modify `build_review_nodes` in `bidlens-engine/src/risk_engine.rs`.

**Current behavior:** For `BlockNode::Table(t)`, the function creates ONE ReviewNode with `table_location: None` and `node_type: ReviewNodeType::TableRow`. The entire table is flattened into one text blob.

**New behavior per D-04:** For `BlockNode::Table(t)`, iterate over `t.rows` and each row's `cells`. Create one ReviewNode per cell with:
- `node_type: ReviewNodeType::TableCell` (confirmed: variant exists at review-core/src/lib.rs:112)
- `table_location: Some(TableLocation { table_index: table_counter, row_index: r, cell_index: Some(c), header_context: vec![] })`
- `original_text`: the cell's content blocks flattened to text (reuse `extract_table_text` logic per cell, or iterate `cell.content` and call `plain_text()` on Paragraph blocks)
- `order_index`: keep using the Traverser's `item.node_index` for the table block, but each cell node gets the same `item.node_index` (or compute a unique one — the node_id is already unique per cell via the hash)

**Implementation approach:**

1. Add a `table_counter: usize` variable before the `for item in Traverser::new(...)` loop, initialized to 0.
2. Replace the `BlockNode::Table(t)` match arm with a loop over `t.rows.iter().enumerate()` and inner loop over `row.cells.iter().enumerate()`.
3. For each cell, extract text by iterating `cell.content` and joining plain_text of Paragraph blocks (handle nested tables by recursing into `extract_table_text` if the content block is a Table).
4. Create a unique `node_id` per cell: use `review_core::generate_node_id(file_hash, &[item.node_index, r, c])` to include table/row/cell indices in the hash input.
5. Build `TableLocation { table_index: table_counter, row_index: r, cell_index: Some(c), header_context: vec![] }`.
6. After the table loop, increment `table_counter += 1`.
7. Use `ReviewNodeType::TableCell` — variant confirmed present at review-core/src/lib.rs:112. No enum changes needed.

**Note on header_context:** Leave as empty vec for now. Header context extraction (first row as column headers) can be added later if needed — it's not in scope for this phase.
