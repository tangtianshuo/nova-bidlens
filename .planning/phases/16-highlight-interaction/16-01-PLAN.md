Create `HighlightOverlay.tsx` — a canvas overlay that renders highlight rectangles and a mouse-tracking tooltip.

**Component signature:**
```typescript
interface HighlightRect {
  x1: number; y1: number; x2: number; y2: number;  // PDF coords
  matchBasis: string;
  similarityScore: number;
  sectionPath: string[];
}

interface HighlightOverlayProps {
  width: number;      // rendered page width in px
  height: number;     // rendered page height in px
  highlights: HighlightRect[];
  pageWidth: number;  // PDF native width (PAGE_DEFAULT_WIDTH)
  pageHeight: number; // PDF native height
}
```

**Implementation details:**

1. **Canvas rendering** — positioned `absolute inset-0` over the PDF page div. `pointer-events: none` on canvas itself so PDF interactions aren't blocked, but we need mouse events for tooltip — use a transparent `div` overlay with `pointer-events: auto` for mouse tracking instead of canvas mouse events.

2. **Drawing highlights:**
   - Scale factor: `width / pageWidth` (maps PDF coords to pixel coords)
   - Each rect: `ctx.fillRect(x1*scale, y1*scale, (x2-x1)*scale, (y2-y1)*scale)`
   - Fill: `rgba(37, 99, 235, 0.3)` (#2563EB at 0.3 opacity) — per user decision D-01
   - Stroke: `1px solid rgba(37, 99, 235, 1)` — per user decision D-01
   - Multi-evidence: different opacity values (0.2, 0.3, 0.4) cycle through highlights — per user decision D-04

3. **Tooltip** — NOT shadcn Tooltip (too heavy for mouse-following on canvas). Use a plain absolutely-positioned div:
   - State: `{ visible: boolean; x: number; y: number; highlight: HighlightRect | null }`
   - On mousemove over the transparent overlay div, check if cursor is inside any highlight rect (using scaled coordinates)
   - If inside: show tooltip div at cursor position (offset 12px right, 12px down)
   - Content: 3 lines — matchBasis text, `相似度: ${(similarityScore * 100).toFixed(1)}%`, `段落: ${sectionPath.join(' > ')}`
   - Styling: `bg-[var(--color-text)] text-[var(--color-bg)] text-xs px-2.5 py-1.5 rounded shadow-md max-w-[280px]`
   - Ensure tooltip stays within overlay bounds (flip to left of cursor if overflowing right)

4. **Redraw on prop changes** — useEffect with `[width, height, highlights]` deps clears and redraws canvas.

5. **Canvas DPR handling** — set canvas element width/height to `width * devicePixelRatio` / `height * devicePixelRatio`, then `ctx.scale(dpr, dpr)`. CSS width/height stays at logical pixels.

  ```typescript
  const pdfHighlights = useMemo(() => {
    if (!selectedFinding || !pdfDrawer.open) return [];
    return selectedFinding.evidence
      .filter(e => {
        const bbox = e.sourceSubmissionId === pdfDrawer.submissionId ? e.sourceBbox : e.targetBbox;
        return bbox && bbox.page > 0;
      })
      .map(e => {
        const bbox = e.sourceSubmissionId === pdfDrawer.submissionId ? e.sourceBbox! : e.targetBbox!;
        return {
          x1: bbox.x1, y1: bbox.y1, x2: bbox.x2, y2: bbox.y2,
          matchBasis: e.matchBasis,
          similarityScore: e.similarityScore,
          sectionPath: e.sourceSubmissionId === pdfDrawer.submissionId ? e.sourceSectionPath : e.targetSectionPath,
        };
      });
  }, [selectedFinding, pdfDrawer.open, pdfDrawer.submissionId]);
  ```
- Pass `highlights={pdfHighlights}` to PdfDrawer
